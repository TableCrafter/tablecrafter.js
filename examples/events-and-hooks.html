TableCrafter.js - Events and Plugin Hooks

Demonstrates the on / off / once events API and lifecycle plugin hooks (beforeEdit / afterEdit, beforeSort / afterSort, beforeLoad / afterLoad, destroy).

1. Events API: on / off / once

table.on(event, handler) returns an unsubscribe function. Use table.once() for a one-shot listener, and table.off(event, handler) to remove by reference. Eight named events are emitted: cellEdit, selectionChange, sort, filter, pageChange, rowAdd, rowUpdate, rowDelete.

Try sorting, filtering, editing a cell, or clicking the buttons below. The log shows every emitted event and its payload.
-- events will appear here --
// All 8 events with their payload shapes:
table.on('cellEdit',        ({ row, field, oldValue, newValue }) => { ... });
table.on('selectionChange', ({ selectedRows })                  => { ... });
table.on('sort',            ({ sortKeys })                      => { ... });
table.on('filter',          ({ filters })                       => { ... });
table.on('pageChange',      ({ page })                          => { ... });
table.on('rowAdd',          ({ row, index })                    => { ... });
table.on('rowUpdate',       ({ row, index, previous })          => { ... });
table.on('rowDelete',       ({ row, index })                    => { ... });

// Once fires exactly once then auto-removes:
table.once('rowAdd', ({ row }) => console.log('first row ever added:', row));

// on() returns an unsubscribe function:
const unsub = table.on('sort', handler);
unsub(); // stops the listener

// off() removes by reference:
table.off('sort', handler);

2. Plugin Lifecycle Hooks

Plugins declare a hooks object. Any before* hook that returns false cancels the operation -- the corresponding after* hook does not fire and the action is not applied. Errors thrown inside hooks are caught and warned; later hooks still run.

The guard plugin below blocks any edit to the Salary column when the value is negative. Try editing that column with a negative number to see the cancellation in the log.
-- hook calls will appear here --
// Plugin with lifecycle hooks:
const auditPlugin = {
  name: 'audit',
  hooks: {
    // beforeEdit: return false to cancel. Payload: { rowIndex, field, value }
    beforeEdit({ rowIndex, field, value }, table) {
      if (field === 'salary' && value < 0) {
        console.warn('Negative salary blocked');
        return false; // cancel the edit
      }
    },

    // afterEdit fires only when the edit was committed
    afterEdit({ rowIndex, field, oldValue, newValue }) {
      console.log(`${field}: ${oldValue} -> ${newValue}`);
    },

    beforeSort({ field, order }) { console.log('sorting', field, order); },
    afterSort({ field, order })  { console.log('sorted',  field, order); },

    beforeLoad({ source })  { console.log('loading from', source); },
    afterLoad({ data })     { console.log('loaded', data.length, 'rows'); },

    destroy() { console.log('table torn down'); },
  },
};
